# Takeshi's Data Castle: System Design & Architecture

## 1. Tools & Tech Stack
* **Frontend:** React (Vite), TypeScript, Tailwind CSS
* **Backend:** Node.js, Express.js
* **Database:** PostgreSQL (Hosted via Neon)
* **AI & External APIs:** Google Gemini 2.5 Flash SDK, Google Imagen 4.0, NewsAPI

---

## 2. System Workflow
The application operates as a full-stack interactive survival gauntlet utilizing a persistent 5-round loop:
1. **Context & Visual Synthesis:** The Express backend fetches real-time global headlines via NewsAPI, synthesizes them into a surreal gameshow obstacle using LLM generation, and generates a persistent Base64 visual representation of the scene using Imagen.
2. **Resource Allocation:** The player is presented with five randomized items queried from the PostgreSQL database and must draft a survival strategy using their selection.
3. **AI Evaluation:** The strategy, item metadata, and obstacle context are processed by a 3-judge AI panel. The backend parses the LLM output into structured JSON to determine the round outcome and increments the global game state.

---

## 3. Artificial Intelligence Implementation
The system leverages the **Google Gemini Ecosystem** for three distinct architectural roles:
* **Dynamic World-Building (Gemini Flash):** Synthesizing unstructured external data (news) into creative, context-aware challenges via open-ended generation.
* **Visual Synthesis (Imagen):** Generating 1:1 photorealistic, high-fidelity UI assets on the fly based on the LLM's world-building output.
* **Deterministic Evaluation (Gemini Flash):** Acting as a structured rules engine. Through precise prompt engineering, the LLM is constrained to act as three distinct personas (**Master Takeshi**, **Chaos-Chan**, and **The Logic Bot**) and output a rigidly formatted JSON array. This bridges the gap between generative reasoning and the fixed data contracts required by the React frontend.

---

## 4. Data Usage & Strategy
To fulfill the requirement of anchoring AI creativity in a controlled data corpus:
* **The Corpus:** A PostgreSQL database seeded with over 100 unique, pre-generated items.
* **Gameplay Impact:** Item randomization (`ORDER BY random() LIMIT 5`) is offloaded to the database layer rather than generated by the LLM. This grounds the AI's reasoning in a structured schema, forcing the model to contextualize fixed physical properties against a dynamic obstacle to calculate scores.

---

## 5. Key Architectural Decisions
* **Multi-Persona AI Evaluation:** To simulate the subjective chaos of a real Japanese gameshow, the grading prompt forces the LLM to adopt three distinct personas (Master Takeshi, Chaos-Chan, and The Logic Bot). This design mathematically reduces LLM grading variance by averaging the scores against a strict threshold (7/10), while providing highly thematic, entertaining user feedback.
* **Asymmetric World-Building:** A deliberate choice was made to pair rigidly structured data (PostgreSQL inventory items) with highly generative data (LLM-synthesized obstacles). This grounds the AI's reasoning, forcing it to evaluate fixed physical properties against an unpredictable challenge, which makes the gameplay feel like a solvable puzzle rather than a random text generator.
* **Ephemeral Base64 Visual Pipeline:** Because the gameshow relies on single-use, transient obstacle images that are discarded after each round, Imagen's raw byte output is parsed into a Data URI and sent directly within the JSON response. This eliminates the need for persistent cloud storage (e.g., AWS S3) and guarantees synchronous loading of the text and UI assets.
* **Hybrid Mock-Live Architecture:** To ensure high availability and prevent "empty state" failures, the system was designed with a dual-layer orchestration strategy. The backend detects missing environment variables or API outages in real-time and seamlessly hot-swaps to a local mock dataset (headlines, obstacles, and judge verdicts). This allows the application to remain fully functional and interactive even in local-offline or zero-quota environments.
* **Deterministic Data Contracts:** To enforce structural integrity on the unpredictable nature of generative AI, the backend utilizes a custom Regex extraction pipeline. This safely strips conversational wrappers and enforces strict JSON arrays, ensuring the React UI always receives predictable object shapes regardless of the LLM's formatting tendencies.
* **Thematic Graceful Degradation:** Rather than breaking immersion with standard HTTP error boundaries, the system translates API rate limits or AI safety filter blocks into in-universe events. For example, failed Imagen generations immediately render a stylized "Visual Feed Offline" UI state, preserving the high-energy aesthetic while keeping the application fault-tolerant.
* **Deterministic QA Override Switch:** Testing stochastic LLM outcomes is notoriously difficult. A hardcoded master passphrase (`CASTLE QA OVERRIDE: CLEAR THIS ROUND`) was built into the evaluation pipeline to safely bypass API calls. This allows for rapid QA testing of the 5-round progression loop without consuming API quotas or waiting on network latency.
